核心特性

copy



| 特性 | 说明 |
| ---- | ------------------ |
| 增量解析 | 只重新解析修改的部分，极快 |
| 实时解析 | 边写代码边解析，毫秒级反馈 |
| 语言支持 | 通过 Grammar 定义支持新语言 |
| 错误恢复 | 语法错误也能生成部分树 |

1. Grammar 定义（以 JS 为例）

JavaScript


// grammar.js
module.exports = grammar({
name: 'javascript',

rules: {
// 程序由多个语句组成
program: $ => repeat($.\_statement),

    // if 语句
    if_statement: $ => seq(
      'if',
      '(',
      $._expression,
      ')',
      $.block,
      optional($.else_clause)
    ),

    // 表达式
    _expression: $ => choice(
      $.binary_expression,
      $.identifier,
      $.number,
      // ...
    ),

    // 二元运算
    binary_expression: $ => seq(
      $._expression,
      $._operator,
      $._expression
    ),

    _operator: $ => choice('+', '-', '*', '/', '>', '<', '==', '==='),

    // ...

}
})

增量解析原理

传统解析 vs Tree-sitter

copy


传统解析：
代码修改 ──► 重新解析整个文件 ──► O(n)

Tree-sitter：
代码修改 ──► 计算差异 ──► 只更新变化的节点 ──► O(k) k=变化区域

具体流程

copy



1. 旧语法树
   (A)
   / \
   B C

2. 修改了 C
   (A)
   / \
   B C'

3. Tree-sitter 智能复用
   (A) ← 保留
   / \
   B C' ← 只更新 C
